9. Generické typy, lambda výrazy

Rozhraní (interface)

  • jiný způsob abstrakce než dědičnost
  • obsahuje pouze hlavičky metod, bez implementace metod
  • u metod se nepíše public abstract (všechny je mají v základu)
  • lze implementovat více rozhraní (na rozdíl od dědičnosti)
  • třída implementující rozhraní musí implementovat všechny metody rozhraní
  • proměnné typu rozhraní mohou odkazovat na libovolný objekt, který toto rozhraní implementuje.
IAuto auto = new Opel();

IAuto auto2;
auto2 = new Opel();

Lambda výrazy

DOPLNIT //funkcionalni rozhrani KONEC

  • zkrácený způsob zapisování anonymních funkcí
  • skládá se z:
    • seznamu parametrů
    • ->
    • těla výrazu

Syntaxe

//pokud je potřeba jen jeden řádek, třeba volání metody
(parametry) -> IO.println("Ahoj světe! Zdraví " + parametry..) 

//pokud je více řádků
(parametry) -> { IO.println("Ahoj světe! Zdraví " + parametr1) 
									IO.pintln("Ahoj světe! Zdraví " + parametr2)..}
									
//pokud je jen 1 proměnná
x -> IO.println("Ahoj světe! Zdraví " + parametr1) 

//pokud není proměnná
parametr1 -> výraz 
() -> IO.println("Ahoj světe!") 

//1. příklad v praxi - forEach
numbers.forEach(x -> System.out.println(x))

//2. příklad v praxi - vlákna
Thread t = new Thread(() -> System.out.println("Vlákno běží"));

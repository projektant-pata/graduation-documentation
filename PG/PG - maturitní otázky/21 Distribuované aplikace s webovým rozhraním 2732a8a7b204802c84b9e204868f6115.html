21. Distribuované aplikace s webovým rozhraním

HTTP protokol

= HyperText Transfer Protocol

  • bezstavový (nepamatuje si předešlé požadavky
  • request-response komunikace
    • klient zasílá požadavek
    • server zasílá odpověď

Metody

GET

  • ukládá data do hlavičky
  • jen hlavička

POST

  • ukládá data do těla
  • hlavička a tělo

Webová komunikace v Javě

  • k hostování Java aplikací (server-side) se využívají hostitelské služby pro JVM
    • nejběžněji Apache Tomcat (Java server se servlet kontejnery)
  • lze provozovat v:
    • čisté Javě
    • frameworkcích (např. SpringBoot)

Struktura webové aplikace

Backend

  • všechna logika
  • databáze

Servlety

  • nejsou potřeba, pokud není dynamický obsah

    ⇒ bez nich dynamický obsah nejde

  • prostředník mezi backendem a frontendem
  • spojuje Javu a HTML

Frontend

  • k dynamickému obsahu se využívají servlety
    • psaním přímo do kódu
    • JSP (kombinace Javy a HTML) - modernější

Životní cyklus servletu

  • plně řízen servletovým kontejnerem

1. Narození

  • třída se načte do mezipaměty, dle konfigurace
    • buď na začátku aplikace
    • při či při prvním zavolání (default)
  • přidá se mapování

2. Příprava na vyběhnutí

  • kontejner si automaticky zavolá u servletu metodu init(ServletConfig conf)
    • ta připraví servlet na běh

3. Běh

  • servlet běží a přijímá požadavky
  • když servletu přijde požadavek, zavolá service(), přepošle dle metody do
    • doGet(req, res)
    • doPost(req, res)
  • ke zpracování požadavku a odpovědi se vytváří objekty HttpServletRequest a HttpServletResponse
  • jsou pouze na 1 požadavek, při dalším se nahradí čistými

HttpServletRequest

  • obsahuje všechna data poslaná serveru
    • HTTP metoda (getMethod())
    • URL/HTTP hlavičku a tělo (getRequestUrl(), getHeader(), getParameterMap())
    • cookies, session.. (getCookies(), getSession())

HttpServletResponse

  • slouží k vytvoření odpovědi
  • součásti je i
    • status kód
      • 200 - OK
      • 403 - Forbidden
      • 404 - Not Found
    • HTTP hlavičku a tělo
    • Cookies
  • užitečné metody jako addCookie(String, Obj), sendDirect(String)

Přesměrování

  • lze docílit díky sendRedirect(String)
  • v hlavičce je status kód 302 a místo k přesměrování
  • zahodí se stará data a vytvoří se nový požadavek

4. Smrt

  • nastane zavoláním destroy()
  • volá se:
    • vypnutí/restart aplikace
    • manuálně

JSP

  • překládá na servlet
  • HTML kód s
    • Expression language (čtení dat z předaných proměnných)
    • Standart Tag Library (smyčky, podmínky..)

Ukázky

// deklarace servletu 
@WebServlet("/muj-servlet")
@WebServlet("/muj-servlet")

//Ukázka JSP
<!-- Výpis proměnné -->
<h1>Vítej, ${jmenoUzivatele}</h1>

<!-- Podmínka -->
<c:if test="${vek >= 18}">
    <p>Jsi plnoletý.</p>
</c:if>

<!-- Cyklus -->
<ul>
    <c:forEach items="${seznam}" var="polozka">
        <li>${polozka}</li>
    </c:forEach>
</ul>

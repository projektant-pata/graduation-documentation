17. Síťová komunikace - TCP a UDP

  • propojení 2 a více zařízení za účelem komunikace
  • na 4 vrstvě - aplikační (z hlediska PS. ne architektury aplikace)
  • využívá se pro Klinet-server architekturu

Základní princip

Klient

  • navazuje spojení jako 1.
  • posílá požadavky

Server

  • čeká na požadavky klientů
  • zpracuje je a posílá odpovědi

Síťový protokol

  • určuje pravidla komunikace (TCP, UDP, RMI..)

Typický postup

1. Navázání spojení

  • klient se připojí podle IP adresy a portu
  • TCP - naváže spojení s handshake
  • UDP - nenaváže spojení, jen se posílají pockety

2. Odeslání požadavků

  • klient posílá data (zpráva..) serveru

3. Zpracování požadavku

  • server přijme požadavek a provede operaci (rozeslání zprávy všem..)

4. Odeslání odpovědi

  • server vrátí odpověď zpět klientovi

5. Ukončení spojení

  • TCP - spojení je třeba uzavřít (FIN/ACK)
  • UDP - není spojení = není co uzavírat

Socket

  • koncový bod obousměrného komunikačního spojení mezi 2 aplikacemi běžícími v síti
  • má IP a číslo portu
    • = vymezený komunikační bod v síti

ServerSocket

  • možnosti inicializace
    • ServerSocket(int port) (port)
    • ServerSocket(int port, int backlog) (port + velikost fronty pro čekající připojení)
    • ServerSocket(int port, int backlog, InetAddress bindAddr) (port + velikost fronty pro čekající připojení + určí přesné síťové rozhraní)
  • socket pro navázání spojení u TCP
  • zavolá se metoda accept() - po připojení uživatele vrací objekt Socket , se kterým komunikuje s klientem
  • nutno uzavřít přes close() (v try-with-resources se uzavře automaticky)

Socket

  • možnosti inicializace
    • Socket(InetAddress address, int port) (adresa a port) - u klienta
    • vrácená hodnota u ServerSocket - u serveru
  • komunikace klienta a serveru v TCP
  • klient si ho vytvoří přes Socket(InetAddress address, int port)
  • server si ho vytvoří přes ServerSocket
  • komunikuje se přes Stream
    • čtení: InputStream - BufferedReader (pro textová data)
    • odesílání: OutputStream - PrintWriter(pro textová data)
  • nutno uzavřít přes close() (v try-with-resources se uzavře automaticky)

DatagramSocket

  • možnosti inicializace
    • DatagramSocket() (random port)
    • DatagramSocket(int port) (port)
    • DatagramSocket(int port, InetAddress ipAddress) (port + IP)
  • komunikace v UDP
  • posílá se DatagramPacke , který obsahuje
    • payload (data)
    • adresu a port cíle
  • send(DatagramPacket packet)
  • receive(DatagramPacket packet)
  • nutno uzavřít přes close() (v try-with-resources se uzavře automaticky)

TCP x UDP

TCP

Plusy

  • spolehlivý přenos
    • zajišťuje doručení dat
    • kontrolu chyb
    • správné pořadí packetů

Mínusy

  • rychlost oproti UDP

Vhodné pro

  • přenos dat, kde záleží na správnosti
    • e-mail
    • chatovací aplikace
    • webové stránky
    • FTP, SSH

Využití v Javě

  • server vytvoří ServerSocket
  • po připojení uživatele na ServerSocket si server udělá Socket

UDP

Plusy

  • rychlost

Mínusy

  • nespolehlivost

Vhodné pro

  • rychlosti dat
    • online hry
    • streamovací platformy
    • multicast

Využití v Javě

  • server i klient si vytvoří DatagramSocket, přes který si posílají DatagramPacket
  • DatagramPacket obsahuje
    • payload (data)
    • adresu a port cíle

Příklady

TCP

Server TCP

try (ServerSocket serverSocket = new ServerSocket(9090)) {
            System.out.println("Server je spuštěn a čeká na klienta...");
            try (Socket clientSocket = serverSocket.accept();
                 BufferedReader in = new BufferedReader(new InputStreamReader(clientSocket.getInputStream()));
                 PrintWriter out = new PrintWriter(clientSocket.getOutputStream(), true)) {

                System.out.println("Klient je připojen!");

                String message = in.readLine();
                System.out.println("Klient: " + message);

                out.println("Zpráva přijata serverem.");
            }
        } catch (IOException e) {
            e.printStackTrace();
	          }

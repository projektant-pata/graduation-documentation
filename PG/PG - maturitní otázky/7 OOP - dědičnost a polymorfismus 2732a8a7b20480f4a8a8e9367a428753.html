7. OOP - dědičnost a polymorfismus

Dědičnost

  • umožňuje potomkovi zdědit všechny atributy a třídy rodiče
  • pro dědění se užívá extends
  • lze dědit jen z jedné třídy
  • super() - odkázání na konstruktor předka
  • super. - odkázání na proměnnou/metodu předka (musí být protected)
  • jakýkoliv potomek může zastoupit svého předka
  • absolutní předek = Object
    • dědí se od něj
      • konstruktor bez parametru
      • metody toString(), equals(), notify()...

Polymorfismus

  • umožňuje, aby různé objekty reagovali na stejnou metodu různými způsoby
  • funguje na principu překrytí - @override (přepíše původní logiku)

Abstrakce a final

Ve zkratce

AbstractFinal
Proměnná❌= konstanta (nelze změnit)
Metodanemá tělo → musí být implementována v potomkovinelze @override
Třídanelze instancovat → nutnost potomka pro implementacikastrace (nemůže mít potomky)

Interface

= rozhraní

  • sada definic, co musí třída splňovat přes implements
  • může obsahovat metody
    • s tělem
      • defualt - není potřeba přepisu
      • static
      • private
    • bez těla
      • abstact
  • může obsahovat vlastnosti (konstanty), pokud jsou
    • static
    • public
    • final
  • taky nemusí nic obsahovat

Funkcionální rozhraní

  • anotace @FunkcionalInterface (volitelně)
  • obsahuje jednu abstraktní metodu (neabstraktních může mít více)
  • může být zastoupeno lambda výrazem
  • standardní funkcionální rozhraní
    • Function<T, R> - převede typ T na R
    • Predicate<T> - test podmínky (true/false)
    • Consumer<T> - přijímá hodnotu, nic nevrací
    • Suplier<T> - nepřijímá nic, vrací hodnotu

Ukázka

Ukázka polymorfismu

abstract class Zvire{
	abstract void zvuk();
}

class Kocka extends Zvire{
	@override
	void zvuk(){
		IO.println("Mnau");
	}
}

class Kapr extends Zvire{
	@override
	void zvuk(){
		IO.println("");
	}
}

Ukázka rozhraní

public interface IZvire(){
	default void zvuk(){
		IO.println("Zvuk");
	}
}

11.-16. - práce s DB

Co to je DB?

Databáze (DB)

  • struktura dat pro uchování záznamů
  • navrhnuta pro vyhledávání a správu dat

Systém řízení báze dat (SŘBZ)

  • program. který
    • zpracovává data, opravuje data z databáze
    • má uživatele
    • obsahuje přístupová data ⇒ ochrana dat

DB systém

  • spojení DB a SŘBZ
  • MySQL, MangoDB, PostgresSQL

Druhy připojení DB

VlastnostPDO MySQLi
Podpora databázíUniverzální (např. PostgreSQL)MySQL, MariaDB
Styl zápisuObjektový Objektový, Procedurální
Prepared StatementsPodporuje pojmenované parametry (:id, :email)Pouze otazníky (?), pojmenované nepodporuje
Získání dat (Fetch)FlexibilnějšíMéně flexibilní, často problém s mapováním na objekt

PDO

  • objek(tivně/tově) lepší
  • budu zmiňovat v příkladech

3 kroky s prací s DB

1. Connection (připojení)

  • vstupní parametry
    • DSN (data source name) - cesta a název DB
    • uživatel - uživatelské jméno v DB
    • heslo - heslo k uživateli v DB
    • pole možností
      • ATTR_ERRMODE
        • co má dělat při špatném dotazu (např překlep)
        • ERRMODE_EXCEPTION (best practice, vrátí výjimku)
      • ATTR_DEFAULT_FETCH_MODE
        • jak mají být výsledná data zpracováná
        • základní módy:
        KonstantaVýstup
        FETCH_BOTH (default)vrací data 2x, jako asoc. pole a jako pole s číselnými indexy
        FETCH_OBJvrací data jako anonymní objekt
        FETCH_CLASSvrací data jako instanci existující třídy (automaticky namapuje)
        FETCH_ASSOCvrací data jako asociativní pole
        FETCH_NUMvrací data jako pole s číselnými indexy
      • ATTR_EMULATE_PREPARE
        • zpracuje dotaz DB či PHP
        • best practice je false (zabraňuje SQL Injection)

Příklad

$options = [
	PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
	PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
	PDO::ATTR_EMULATE_PREPARE => false
];
$pdo = new PDO(
        'mysql:host=127.0.0.1;dbname=test;charset=utf8mb4', // 1. DSN
        'root',                                             // 2. Uživatel
        '',                                                 // 3. Heslo
        $options                                            // 4. Options
    );

2.1.1. Prepare (příprava dotazu)

  • udělá se dotaz bez parametrů (ty se dosadí až při execute)
  • 2 typy dosazení
    • pojmenované parametry (best practice)
    • otazníkové parametry (starší)

Ukázka

//pojmenovane param
$sql1 = "SELECT * FROM uzivatel WHERE vek = :vek AND pohlavi = :pohlavi"
$stmt1 = $pdo->prepare($sql1);

//otaznikove param
$sql2 = "SELECT * FROM uzivatel WHERE vek = ? AND pohlavi = ?"
$stmt2 = $pdo->prepare($sql2);

2.1.2. Execute (provedení dotazu)

  • dosadí se parametry a provede se dotaz

Ukázka

$data = [
	'vek' => 69,
	'pohlavi' => true //pohlavi by melo byt vzdy boolean ['man' => true, 'woman' => false, 'jine' => null]
];

$stmt1->execute($data);

//NEBO pres nabindovani parametru 
// => pri zmene hodnoty promene se zmeni vyhledavaci parametr
$stmt1->bindParam(':vek', $data['vek']);
$stmt1->bindParam(':pohlavi', $data['pohlavi']);
$data['pohlavi'] = false;
$stmt1->execute(); //provede WHERE pohlavi = false

//NEBO pres nabindovani value, ne promena (zmena hodnoty promene nema efekt)
// => nastavila se hodnota, ne 
$stmt1->bindValue(':vek', $data['vek']);
$stmt1->bindValue(':pohlavi', $data['pohlavi']);
$data['pohlavi'] = false;
$stmt1->execute(); //provede WHERE pohlavi = true

//NEBO nabindovani dat az v hlavicce metody
$stmt2->execute([69, true]);

2.2. Query

  • pokud dotaz nepotřebuje proměnné od uživatele
  • POKUD TAM BUDOU HROZÍ SQL INJECTION

Ukázka

$stmt = $pdo->query("SELECT * FROM uzivatel");

3. Fetching

  • vytáhne data ze statementu
  • jen pro SELECT

fetch()

  • vrací 1 řádek

fetchAll()

  • vrací všechny řádky jako pole
$user = $stmt->fetch(); //ziskane prvniho uzivatele

//NEBO
$users = $stmt->fetchAll();

Další užitečné metody

  • fetchColumn() - vrací hodnotu jednoho sloupce z dalšího řádku
  • rowCount() - počet ovlivněných řádků (u INSERT, UPDATE, DELETE)
  • lastInsertId() - poslední autoincrementované ID

22. Primární klíč, cizí klíč, normalizace

Klíče

Primární klíč (PK)

  • unikátní identifikátor
  • typy
    • jednoduchý - jeden atribut
    • složený - více atributů

Cizí klíč (FK - foreign key)

  • atribut obsahující PK jiného záznamu (může být ve stejné tabulce)

Normalizace

  • proces organizace dat za účelem snížení redundance a zajištění integrity

SQL, noSQL

SQL (Structured query language)

  • dotazovací jazyk
  • deklarovací jazyk jako HTML (popisuje se hotový výsledek, ne cesta k němu!!)
  • struktura dat je ve formě tabulek s pevně definovaným schématem (sloupce + dat. typy)

noSQL (not only SQL)

  • DB pro flexibilitu struktury dat
  • lepší škálovatelnost dat a vyšší výkon
  • ukládají se dokumenty (JSON, BSON, k-v (klíč value), grafy)
  • schéma není nebo hodně flexibiliní
  • nemá SQL dotaz. jazyk, ale třeba noSQL - JSON(JS), MongoDB (JSON like syntax)

Škálovatelnost

  • SQL - vertikální škálovatelnost ⇒ potřeba výkonnějšího serveru
  • noSQL - horizontální škálovatelnost ⇒ na více serverech ⇒ možnost většího objemu dat

Typy noSQL DB

  • dokumentová DB
    • ukládá jako dokumenty (JSON, BSON..)
    • př. MongoDB
  • K-V
    • ukládá data jako hodnoty K-V
    • př. Redis
  • sloupcová DB
    • data ukládaná po sloupečcích
    • př. Cassandra
  • grafové DB
    • ukladá grafy

  • 3 (5)V (základní charakteristika, tzv. big data)
    • volume (objem) = masivní objem dat /TB-PT/
    • velocity (rychlost) = data vznikají a musí být zpracována velmi rychle /SC/ dc
    • variety (rozmanitost) = různé typy dat (text, obrázek..)
    • veracity (věrohodnost) = kvalita a spolehlivost dat
    • value (hodnota) = schopnost získat užitečné informace

19. PHP - session, cookies

HTTP požadavky

= request a response požadavky

  • v praxi se většinou očekává odpověď (response)
  • stateless = neudržuje stav (až na část mezi request a response)
  • pro nějakou formu state se využívají sessions a cookies

Request x Response

  • request = klient → server
  • response = klient ← server
  • stejná struktura
  • liší se prvním řádkem
  • request - metoda a URL
  • response - verze protokolu + stavový kód

Složení

Hlavička

  • metoda (GET/POST
  • host (doména)
  • cookies
  • user agent (typ a verze prohlížeče a zařízení)

Tělo

  • data

Cookies

  • uložený na straně klienta v prohlížeči
  • k-v databáze

Plusy

  • lze uložit předchozí akce
    • přihlášení, aby se uživatel nemusel pokaždé přihlašovat
    • košík, když třeba uživatel zavře stránku a vrátí se třeba druhý den

Mínusy

  • vždy se pošlou v hlavičce
  • vázané na prohlížeč (některé prohlížeče mají profily, takže se cookies sdílí mezi PC, pokud není profil, tak vázané na PC)
  • lze je ukrást

Nastavení cookies

metoda setcookie()

  • vstupní parametry
    • název (key) - jediný povinný parametr
    • hodnota (value)
    • trvanlivost
      • čas platnosti (je v s)
      • počet sekund od 1. 1. 1970 (v praxi time() + 3600 - pro hodinu platnosti)
      • zrušení cookie - nastavení času do minulosti
      • session cookie (smaže se zavřením prohlížeče) - 0 (výchozí hodnota)
    • cesta
      • / = celá doména
      • /wa-maturitni-otazky = pro /wa-maturitni-otazky a její poddomény
    • domain
      • doména, pro kterou je cookie vytvořena
      • example.com by bylo pro www.example.com
    • secure
      • true/false, cookie se odešle jen při HTTPS
    • httponly
      • true/false přístupná pouze skrz http protokol, ne pro skriptovací jazyky (JS..)
    • samesite
      • zda se cookie posilá mezi weby (cross-site)

Přístup ke cookie

  • $_COOKIES[key] - vrací hodnotu podle klíče
  • hned po uložení není v poli cookies, klientovi se pošle až s response

Session

= “server cookie”

  • váže se na cookie u klienta (ta obsahuje session ID)
  • metoda session_start
    • vytvoří session
    • pošle cookie klientovi s ID session
  • přístup skrz $_SESSION[key]
  • metoda session_destroy
    • zneplatní session
    • nesmaže uživatelskou cookie

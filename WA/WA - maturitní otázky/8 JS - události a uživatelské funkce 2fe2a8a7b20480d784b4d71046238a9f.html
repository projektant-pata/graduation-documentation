8. JS - události a uživatelské funkce

Funkce

  • blok kódu, jež je možné spustit odkudkoli
  • lowkey Objekt
    • některým možno dát proměnné jako objektu
    • některé mají konstruktor new

Deklarace funkce

3 typy:

Klasická deklarace

function nazev(..) {blok}

  • povinný název
  • podporuje hoisting (vloží se na začátek kódu, ať jde volat odkudkoli)
  • this odkazuje na vnitřní blok
  • lze použít konstruktor (new)
  • speciální objekt arguments

Funkční výraz (function expression)

const foo = function(a, b) {blok}

  • jako klasická deklarace, ale
  • nepodporuje hoisting
  • může být anonymní (beze jména)

Lambda (arrow function)

const foo = (a, b) => {blok}

  • osekaná funkce
    • kratší a čitelnější syntax
    • nemá this (bere this z místa, kde je deklarována, proto používat na vlastní nebezpečí)
    • nemá new
    • nemá hoisting

Spouštění funkce se zpožděním / v časovém intervalu

setTimeout() - pokud chceme zpoždění

setInterval() - pokud interval

  • liší se jen tím, že se jedna opakuje a názvem
  • hlavička - (nazevFunkce, param1, param2..)
  • vrací svoje ID
  • lze je zrušit přes clearTimeout(id) a clearInterval(id)

Ukázka

/* ==========================================================================
   1. DEKLARACE FUNKCÍ
   ========================================================================== */

// a) Function Declaration (Klasická)
// ✅ Hoisting (lze volat předem), ✅ má 'this', ✅ má 'arguments', ✅ lze 'new'
function scitej(a, b) {
    console.log("Klasická funkce arguments:", arguments);
    return a + b;
}

// b) Function Expression (Výraz)
// ❌ Bez hoistingu (až po řádku), ✅ má 'this', ✅ má 'arguments'
const nasob = function(a, b) {
    return a * b;
};

// c) Arrow Function (Šipková)
// ❌ Bez hoistingu, ❌ NEMÁ vlastní 'this' (bere z okolí!), ❌ NEMÁ 'arguments', ❌ NELZE 'new'
const odecti = (a, b) => a - b; 
// const odecti = (a, b) => { return a - b; } // delší zápis s blokem

/* ==========================================================================
   2. OBJEKTY A 'THIS' (Klasika vs Arrow)
   ========================================================================== */

const uzivatel = {
    jmeno: "Karel",
    
    // Klasická metoda -> 'this' je tento objekt (uzivatel)
    pozdravKlasicky: function() {
        console.log(`Ahoj, já jsem ${this.jmeno} (Klasika)`);
    },

    // Arrow metoda -> 'this' je to, co je venku (zde window/global), NE objekt!
    // ⚠️ Častá chyba začátečníků
    pozdravArrow: () => {
        console.log(`Ahoj, já jsem ${this.jmeno} (Arrow - chyba!)`); 
    },

    // Arrow v callbacku -> Tady je super! Dědí 'this' z metody 'zpozdenePozdraveni'
    zpozdenePozdraveni: function() {
        setTimeout(() => {
            console.log(`...po vteřině: Jsem ${this.jmeno} (Arrow v callbacku OK)`);
        }, 1000);
    }
};

uzivatel.pozdravKlasicky(); // "Ahoj, já jsem Karel"
uzivatel.pozdravArrow();    // "Ahoj, já jsem undefined" (protože this != uzivatel)
uzivatel.zpozdenePozdraveni(); // Funguje správně díky arrow funkci


function pozdrav(jmeno) {
    console.log("Ahoj " + jmeno);
}

/* ==========================================================================
   3. Timeout a Interval
   ========================================================================== */

// Spustí funkci 'pozdrav' za 2000 ms (2 sekundy) s parametrem "Karel"
let id = setTimeout(pozdrav, 2000, "Karel");

function tikTak() {
    console.log("Tik ťak...");
}

// Spouští 'tikTak' každou vteřinu (1000 ms)
let id = setInterval(tikTak, 1000);

Události, Listenery a Handlery

Jak fungují ve 3 krocích

  1. Něco se stane (kliknutí tlačítka…)
  1. Vznikne Event objekt (objekt s informacemi, co se stalo, kde…)
  1. Spustí se Handler (manipulátor)

Nejčastější události

click - kliknutí myši

submit - odeslání formuláře

keydown - stisk klávesy

DOMContentLoaded - načtení obsahu na stránce (často u scriptu, lze nahradit s parametrem defer)

change x input

  • input event se posílá vždy po napsání charakteru
  • change event se pošle až po změny focusu inputu (dokončení/kliknutí vedle)
  • change jde u checkboxů a selectů, input jen u text-like inputů

Listenery a Handlery

Listener = část, co poslouchá pro event a spustí handler

Handler = všechna logika, co se má vykonat při té události

2 způsoby:

On..

  • např. onclick, onload
  • k jednomu eventu lze přiřadit 1 funkce
  • zastaralé

v HTML

<button onclick="kliknuto()">Klikni mě</button>

v JS

  • lepší jak v HTML (oddělení logiky od frontendu bo grafici)
const btn = document.querySelector("button");
btn.onclick = () => console.log("Klik!");

addEventListener

  • nové možnosti
    • více event listenerů na 1 event
    • parametry a možnosti navíc (ukázka základních
      • once - vykoná se jen jednou, pak se blokne
      • capture - změna směru šíření (z probublávání na kapťuru)
  • moderní přístup
const btn = document.querySelector("button");

btn.addEventListener("click", () => {
    console.log("První akce");
});

btn.addEventListener("click", (event) => {
    console.log("Druhá akce");
},{
  once: true,
  passive: true
}););
